# Naming conventions

## General rule

Names are part of the design.

A misleading name is a design defect.

## Domain vocabulary

Use the vocabulary of the problem domain.

Examples:

Good:
- LegacyProject
- MigrationPlan
- AudacitySession

Avoid:
- OldFile
- Thing
- DataObject

## Rename when concepts evolve

Do not preserve obsolete names for compatibility inside the codebase.

When a concept changes, its name must change too.
